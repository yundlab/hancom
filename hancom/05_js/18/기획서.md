# 📒 미니 프로젝트 기획서

## 여행 소비 기록장 — 트래블틱 / Travel-Tick

> 작성일: 2026-07-08 · 제작: HTML + CSS + JavaScript · 멀티페이지

---

## 1. 한 줄 소개

여행 중 지출을 적으면 **오늘 환율로 원화까지 자동 계산**해 주고,
**카테고리·메모**와 함께 **여행별로 모아 보고**, 일행과 쓴 돈을
**최소 송금 루트로 정산**까지 해 주는 소비 기록장.

---

## 2. 목표 (왜 만드나)

- 여행지에서 *"이거 우리 돈으로 얼마지?"* 를 매번 계산하기 귀찮음
- 영수증은 버려지고, 그날의 기분·맛집 평가는 금방 잊힘
- 일행과 나눠 낸 돈, *누가 누구에게 얼마 줘야 하는지* 정산이 복잡함

➡️ **기록 + 환율 + 정산**을 한 번에 해결하는 여행 가계부를 만든다.

---

## 3. 핵심 기능

| # | 기능 | 설명 |
|---|------|------|
| 1 | **여행 아카이브** | `26.4 후쿠오카` 처럼 여행 단위로 관리 (진행중 / 예정 / 지난 여행 구분) |
| 2 | **지출 기록** | 카테고리 이모지 · 가게명 · 현지금액 · 날짜 · [개인/공동] · 결제자 · 메모 |
| 3 | **공동지출 참여자 선택** | 공동지출도 전원이 아닌 일부만 나눠 낼 수 있게 참여자를 직접 선택 (기본 전원 → 탭해서 빼기) |
| 4 | **환율 자동 계산** | 환율을 넣으면 현지통화 + 원화(KRW) 동시 표기, 입력 즉시 미리보기 |
| 5 | **예산 대시보드** | 예산 대비 사용률 막대그래프 + 사용률에 따라 바뀌는 응원 문구 |
| 6 | **최적 정산** | 공동지출을 모아 **최소 송금 루트**(그리디 알고리즘) + 인원별 내역(±) 계산 |
| 7 | **여행 · 지출 수정** | 만든 여행 정보(이름·기간·예산·환율·멤버)와 개별 지출 기록 모두 나중에 수정 가능 |

**예시 기록**
> 🍜 이치란 라멘 — 3,200엔 (≈ 28,800원) · [공동 3명] · 결제 나 · 1/3 · *"인생 라멘 등극!"*

## 4. 화면 구성 (멀티페이지 — 파일마다 다른 화면)

페이지끼리는 `localStorage` 로 데이터 공유.

| 페이지 | 파일 | 내용 |
|--------|------|------|
| 🏠 **홈 / 여행 리스트** | `index.html` | 풀스크린 랜딩 히어로(마스코트·워드마크) + 총 여행·지출 통계 + 진행중·지난 여행 카드 + [＋ 새 여행] |
| ✍️ **여정 생성 / 수정** | `create.html` | 나라 선택 → 통화·환율 자동 · 이름 · 기간 · 예산 · 멤버 (`?id=` 로 기존 여행 수정도 겸함) |
| 🧾 **여행 타임라인** | `trip.html` | 여행 기간 표시 + 예산 대시보드 + 지출 기록 폼 + 일차별 리스트 (카드 클릭 시 수정) |
| 💸 **최종 결산** | `report.html` | 여행 제목·국가·기간 + 공동지출 총액 + 최적 송금 루트 + 인원별 내역 |

```
[홈] --선택--> [타임라인] --결산--> [최종 결산]
  |                ^  |
  +--새 여행--> [여정 생성/수정] --만들기/저장--+
```

## 5. 사용 기술 (배운 것)

- **HTML** — 페이지 4개 뼈대(제목·입력칸·버튼·결과칸)
- **CSS**
  - 레트로 캐릭터 테마 — 마스코트 일러스트 · 하프톤(점무늬) 워드마크 · 손그림 두들 장식
  - 카드 레이아웃 · 막대그래프 · 반응형(모바일 1단 → PC 화면 폭 확장)
- **JavaScript**
  - 이벤트 (`addEventListener`) 로 버튼·입력·카드 클릭 반응
  - `querySelector` · `createElement` · `appendChild` 로 화면 갱신
  - **배열 / 객체** 로 여행·지출·멤버 데이터 관리
  - **조건문 · 함수** 로 계산 로직 (환율 · 합계 · 정산 알고리즘 · 일차 계산)
  - **localStorage** 로 저장 + **페이지 간 데이터 공유** (`store.js` 공용 모듈)
  - 주소창 `?id=` 쿼리로 어떤 여행인지 구분 (생성·수정 폼 겸용)
