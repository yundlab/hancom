자기소개서

개발자가 되기로 한 계기

코로나19로 백신 예약 경쟁이 치열하던 시기, GitHub에 공개된 백신 예약 자동화 도구를 보고 개발이 누군가의 불편함을 실질적으로 해결할 수 있다는 걸 처음 실감했습니다. '나도 저런 걸 만드는 사람이 되고 싶다'는 마음이 생겼지만, 그때는 디자인 실무를 이어가고 있었고 바로 뛰어들 용기를 내지 못했습니다. 이후 3년 가까이 웹 디자인·퍼블리싱 업무를 하면서도 그 마음은 사라지지 않았고, 결국 더 미루면 후회로 남을 것 같아 늦었지만 직접 구현하는 개발자의 길을 선택했습니다.

강점

데브코스 팀 프로젝트에서 게시글 목록 화면을 구현하던 중, 데이터가 많아질수록 페이지 로딩이 눈에 띄게 느려지는 문제를 발견했습니다. 처음에는 프론트엔드 렌더링 문제라고 생각했지만, 네트워크 탭을 확인해보니 API 응답 자체가 2초 이상 걸리고 있었습니다. '왜 이렇게 느릴까'라는 질문에서 출발해 백엔드 코드를 직접 살펴봤고, 게시글을 조회할 때마다 댓글 수를 매번 별도 쿼리로 계산하고 있다는 걸 발견했습니다. 팀원들과 상의해 쿼리를 조인 방식으로 개선하고, 필요한 데이터만 페이지네이션으로 나눠 불러오도록 구조를 바꿨습니다. 그 결과 평균 응답 시간을 2초에서 400ms대로 줄일 수 있었고, 이 경험을 통해 화면에 보이는 문제라도 원인은 다른 레이어에 있을 수 있다는 것, 그리고 궁금증을 끝까지 파고드는 태도가 실제 문제 해결로 이어진다는 것을 배웠습니다.

이전 경력

3년간 퍼블리싱과 디자인을 하며 사용자가 실제로 마주하는 화면의 디테일과 접근성을 고민해왔습니다. 이 경험은 지금 프론트엔드 개발을 할 때, 단순히 화면을 구현하는 것을 넘어 사용자 경험까지 고려하는 태도로 이어지고 있습니다.
